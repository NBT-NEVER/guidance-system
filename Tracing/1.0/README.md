# 实验一（中等）：追踪法导引方法—速度比 P 对导引弹道的影响

## 1. 实验任务
这个目录对应《guidance-system.pdf》中的 **实验一（中等）：追踪法导引方法—速度比 P 对导引弹道的影响**。代码直接围绕题目给出的工况做二维平面数值积分，没有再拆额外模块，便于顺着公式核对每一步计算。

- 实验目的：验证速度比 P 改变后追踪法弹道、最大需用过载和脱靶量的变化，并观察 1 < P < 2 这一经验范围。
- 重点扫描量：速度比 P = 0.5，0.8，1.0，1.4，1.5，1.8，2.0，2.2，2.5
- 默认时间步长：`0.01 s`
- 导弹可用过载：`20 g`
- 速度单位：`m/s`

## 2. 建模假设
参考资料中追踪法部分采用的是典型平面追踪模型，这里保留这个前提，只做了三条必要的工程化说明。

- 题目给的是 `Ma`，代码统一按 `1 Ma = 340 m/s` 转换。
- 目标做水平匀速直线运动，默认沿 `x` 轴正方向飞行。
- 对实验 5，姿态回路按参考资料中的二阶环节 `ξ = 0.707, ω = 10` 处理。

## 3. 数学模型
公式展示：

$$
\lambda = \operatorname{atan2}(y_T - y_M, x_T - x_M)
$$

这个式子给出当前时刻的弹目视线角，追踪法的核心就是让导弹速度方向尽量贴着这条视线走。

其中：
- `λ`：目标视线角，单位 `rad`。
- `(x_M, y_M)`：导弹位置，单位 `m`。
- `(x_T, y_T)`：目标位置，单位 `m`。

$$
\dot{\gamma}_c = \frac{\lambda - \gamma}{\Delta t}, \qquad
n_c = \frac{V_M \dot{\gamma}_c}{g}
$$

代码里把“速度矢量指向目标”离散成一条很直接的更新式：先算本步导弹应当转多少角，再换算成需用过载。

其中：
- `γ`：导弹弹道偏角，单位 `rad`。
- `Δt`：积分步长，单位 `s`。
- `V_M`：导弹速度，单位 `m/s`。
- `n_c`：需用法向过载，单位 `g`。

对实验 5 还要加姿态回路：

$$
G_z(s) = \frac{n_y}{n_c} = \frac{\omega^2}{s^2 + 2\xi\omega s + \omega^2}
$$

这里的 `n_y` 是真实过载，`n_c` 是制导律给出的指令过载。没有姿态回路时，两者默认直接相等；有姿态回路时，真实过载会慢一拍。

LaTeX 代码：

```latex
\lambda = \operatorname{atan2}(y_T - y_M, x_T - x_M)
\dot{\gamma}_c = \frac{\lambda - \gamma}{\Delta t}
n_c = \frac{V_M \dot{\gamma}_c}{g}
G_z(s) = \frac{n_y}{n_c} = \frac{\omega^2}{s^2 + 2\xi\omega s + \omega^2}
```

## 4. 数值求解流程
程序没有把主循环再拆成很多文件，核心步骤都写在 `main.py` 里，顺着代码可以直接看到下面这条流程。

1. 根据题目给出的初始弹目距离、初始视线角和速度设置导弹与目标初值。
2. 在每个时间步先算当前视线角 `λ`，再求导弹本步需要转过的角速度。
3. 把角速度换成需用过载，并按 `20 g` 做实际过载约束。
4. 用更新后的弹道偏角推进导弹位置，同时推进目标位置。
5. 记录最大需用过载、最大实际过载、最小弹目距离和命中标记。
6. 扫描结束后输出 `summary.csv`、`summary.json` 和图形文件。

## 5. 代码结构
```text
.
├─config.py
├─main.py
├─README.md
└─requirements.txt
```

- `config.py`：统一管理路径、输出文件名和题目参数。
- `main.py`：完成追踪法积分、过载统计和作图。
- `README.md`：说明题目、公式、流程和输出文件。
- `requirements.txt`：列出 `numpy` 与 `matplotlib`。

调用关系很简单：`main.py -> build_config() -> run_tracking_case() -> 保存 CSV/JSON/PNG`。

## 6. 运行方式
在当前目录执行：

```bash
python main.py
```

## 7. 输出文件
程序运行后会在 `outputs/` 下生成：

- `tables/summary.csv`：每个扫描工况的最大需用过载、最大实际过载、脱靶量、命中标记。
- `tables/summary.json`：同一批结果的结构化版本。
- `figures/trajectory.png`：弹道对比图。
- `figures/metrics.png`：需用过载和脱靶量曲线。
- `figures/attack_zone.png`：只有攻击区实验会额外生成这张图。

## 8. 本实验的实现对应关系
- 当前实验模式：`speed_ratio_sweep`
- 题目要求中的核心观察量：不同速度比下的最大需用过载、脱靶量，以及哪些工况会因为 20g 约束而失去可实现性。
- 如果你想改扫描密度，直接在 `config.py` 里的列表字段里改，不需要改主循环结构。
