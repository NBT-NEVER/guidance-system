# 实验二（中等）：比例导引法—导航比对导引弹道的影响（噪声）

## 1. 实验任务
这个目录对应《guidance-system.pdf》中的 **实验二（中等）：比例导引法—导航比对导引弹道的影响（噪声）**。实现思路按教材里的比例导引主线展开：先算视线角速度，再由导航比得到法向过载指令。

- 实验目的：在比例导引里加入视线角速率白噪声，比较不同导航比和噪声水平下的弹道稳定性。
- 重点扫描量：导航比与噪声组合 = K=3,4,6，σ=|λ̇0|/30, |λ̇0|/20, |λ̇0|/10
- 默认时间步长：`0.01 s`
- 导弹可用过载：`20.0 g`
- 命中阈值：`0.1 m`

## 2. 建模假设
- 比例导引按二维平面点质点模型实现，速度大小保持常值，只改变速度方向。
- 目标机动实验采用“法向定常过载”方式更新目标航向角。
- 噪声实验中，原 PDF 对“方差”描述没有写清单位，代码按“视线角速率标准差取初值的 1/30、1/20、1/10”处理，这样量纲是一致的。
- 姿态回路采用参考资料给出的二阶环节 `ξ = 0.707, ω = 10`。

## 3. 数学模型
公式展示：

$$
\dot{\lambda} = \frac{x_r \dot{y}_r - y_r \dot{x}_r}{r^2}
$$

这个式子是比例导引实现里最关键的一步，视线角速度直接决定导弹需要给多大的法向转弯。

其中：
- `r = \sqrt{x_r^2 + y_r^2}`：弹目距离。
- `(x_r, y_r)`：目标相对导弹的位置分量。
- `(\dot x_r, \dot y_r)`：目标相对导弹的速度分量。

$$
V_c = -\frac{x_r \dot{x}_r + y_r \dot{y}_r}{r}, \qquad
a_c = K V_c \dot{\lambda}
$$

这里 `V_c` 是接近速度，`K` 是导航比，`a_c` 是比例导引给出的法向加速度指令。

对噪声实验：

$$
\dot{\lambda}_m = \dot{\lambda} + w
$$

其中 `w` 是视线角速率测量噪声。

对姿态回路实验：

$$
G_z(s) = \frac{n_y}{n_c} = \frac{\omega^2}{s^2 + 2\xi\omega s + \omega^2}
$$

LaTeX 代码：

```latex
\dot{\lambda} = \frac{x_r \dot{y}_r - y_r \dot{x}_r}{r^2}
V_c = -\frac{x_r \dot{x}_r + y_r \dot{y}_r}{r}
a_c = K V_c \dot{\lambda}
\dot{\lambda}_m = \dot{\lambda} + w
G_z(s) = \frac{n_y}{n_c} = \frac{\omega^2}{s^2 + 2\xi\omega s + \omega^2}
```

## 4. 数值求解流程
1. 初始化目标和导弹位置、速度、导航比与过载上限。
2. 在每个时间步先计算弹目相对位置和相对速度。
3. 用相对运动几何关系计算视线角速度 `λ̇` 与接近速度 `V_c`。
4. 由 `a_c = K V_c λ̇` 计算需用过载，并按可用过载上限做裁剪。
5. 若题目要求加入噪声或姿态回路，就在这一步插入噪声测量和二阶环节。
6. 更新导弹弹道倾角、推进弹目位置，记录最大需用过载、脱靶量和命中情况。
7. 试验结束后输出表格和图形。


- `config.py`：统一保存场景、扫描量和输出路径。

## 6. 运行方式
```bash
python main.py
```

## 7. 输出文件
- `outputs/tables/summary.csv`：导航比、速度比、机动过载或发射点扫描结果。
- `outputs/tables/summary.json`：对应的结构化摘要。
- `outputs/figures/trajectory.png`：典型弹道图。
- `outputs/figures/metrics.png`：最大需用过载和脱靶量曲线。
- `outputs/figures/attack_zone.png`：实验 5 的允许攻击区图。

## 8. 当前实验模式
- 模式：`noise_sweep`
- 题目要求中的核心观察量：这项实验不是只看 K，还要看测量噪声会不会放大过载需求和脱靶量。
- 需要改扫描密度时，优先改 `config.py` 中的列表，不要先动 `main.py` 主循环。

## 9. 说明
这一组实验里，比例导引的核心公式没有做额外改写，主要是在代码层面把“视线角速度、接近速度、过载裁剪、姿态回路”四个步骤拆开写清。这样做的好处是后面写实验报告时，能直接把公式和代码段一一对应起来。
