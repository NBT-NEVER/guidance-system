# 实验 2.0：追踪法中初始视线角对拦截过程的影响

## 1. 实验内容

本实验研究纯追踪法中，目标初始位置对应的初始视线角 \(q_0\) 变化后，对拦截弹道形状、需用过载和脱靶量的影响。导弹始终把速度方向转向目标瞬时所在方向，因此该实验可以直接反映纯追踪法对几何初始条件的敏感性。

本版本只完成“初始视线角扫描”这一项内容，不包含其他实验的接口、模式切换或预留函数。

## 2. 文件组成

```text
2.0

```

- `config.py`：保存本实验单独使用的参数、路径和输出文件名。

- `requirements.txt`：列出运行依赖。

## 3. 模型与公式

### 3.1 初始几何关系

导弹初始位置设为原点，目标初始位置由初始距离 \(R_0\) 和初始视线角 \(q_0\) 给出：

\[
x_T(0)=R_0\cos q_0,\qquad y_T(0)=R_0\sin q_0
\]

导弹初始航向取为视线方向：

\[
\gamma(0)=q_0
\]

### 3.2 纯追踪法控制思想

纯追踪法要求导弹速度方向始终指向目标当前位置，因此瞬时指令航向角为：

\[
\gamma_c=\operatorname{atan2}(y_T-y_M,\ x_T-x_M)
\]

航向误差为：

\[
e_\gamma=\gamma_c-\gamma
\]

程序中把角度限制在 \([-\pi,\pi]\) 区间内，避免跨越 \(\pm\pi\) 时出现跳变。

### 3.3 过载指令

离散步长为 \(\Delta t\) 时，航向角速度指令近似写成：

\[
\dot{\gamma}_c \approx \frac{e_\gamma}{\Delta t}
\]

对应法向过载指令为：

\[
n_c=\frac{V_M\dot{\gamma}_c}{g}
\]

实际过载受限于最大过载：

\[
n=\operatorname{sat}(n_c,\ -n_{\max},\ n_{\max})
\]

### 3.4 运动学积分

导弹运动方程：

\[
\gamma_{k+1}=\gamma_k+\frac{n g}{V_M}\Delta t
\]

\[
x_{M,k+1}=x_{M,k}+V_M\cos\gamma_k\Delta t
\]

\[
y_{M,k+1}=y_{M,k}+V_M\sin\gamma_k\Delta t
\]

目标按匀速直线运动：

\[
x_{T,k+1}=x_{T,k}+V_T\cos\psi_T\Delta t
\]

\[
y_{T,k+1}=y_{T,k}+V_T\sin\psi_T\Delta t
\]

## 4. 程序流程

1. 读取 `config.py` 中的初始视线角数组和其余固定参数。
2. 对每个 \(q_0\) 单独建立一组导弹与目标初始状态。
3. 在 `main()` 中直接进行逐步积分。
4. 每一步计算目标相对位置、指令航向角、航向误差、需用过载和实际过载。
5. 更新导弹与目标坐标，并记录最小距离、飞行时间、过载饱和比例。
6. 输出汇总表和图像。

## 5. 输出结果

运行结束后会在 `outputs` 下生成：

- `tables/summary.csv`：汇总各初始视线角下的最大需用过载、最大实际过载、脱靶量、是否命中、飞行时间、饱和比例。
- `tables/summary.json`：与 CSV 对应的结构化结果。
- `figures/trajectory.png`：不同初始视线角下的导弹轨迹图。
- `figures/metrics.png`：初始视线角与最大需用过载、脱靶量的关系图。

## 6. 运行方法

```bash
python main.py
```

## 7. 结果观察重点

- 当初始视线角增大时，导弹初始转向需求通常更强，需用过载会明显变化。
- 某些初始视线角条件下，纯追踪法会出现较大弯曲弹道，飞行路径变长。
- 若需用过载长期超过上限，实际轨迹会偏离理想追踪轨迹，脱靶量可能增大。
