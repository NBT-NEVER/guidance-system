# 实验一（中等）：追踪法导引方法—速度比 P 对导引弹道的影响

## 1. 实验内容
本实验只研究一个问题：当导弹速度与目标速度之比 `P` 改变时，追踪法导引弹道、最大需用过载和脱靶量会怎样变化。

- 目标速度：`0.8 Ma`
- 导弹速度：按 `P × 目标速度` 计算
- 初始弹目距离：`4000 m`
- 初始目标视线角：`90°`
- 导弹最大可用过载：`20 g`
- 时间步长：`0.01 s`

扫描的速度比为：

```text
P = 0.5, 0.8, 1.0, 1.4, 1.5, 1.8, 2.0, 2.2, 2.5
```

## 3. 计算模型

追踪法的核心做法是让导弹速度方向始终朝向目标当前位置。当前时刻弹目视线角写成：

$$
\lambda = \operatorname{atan2}(y_T - y_M, x_T - x_M)
$$

其中：

- `λ`：弹目视线角
- `(x_M, y_M)`：导弹位置
- `(x_T, y_T)`：目标位置

导弹本步需要修正的弹道角速度按离散形式写成：

$$
\dot{\gamma}_c = \frac{\lambda - \gamma}{\Delta t}
$$

再换算为需用法向过载：

$$
n_c = \frac{V_M \dot{\gamma}_c}{g}
$$

程序里再对 `n_c` 做 `20 g` 限幅，得到真实使用的法向过载。

LaTeX 代码：

```latex
\lambda = \operatorname{atan2}(y_T - y_M, x_T - x_M)
\dot{\gamma}_c = \frac{\lambda - \gamma}{\Delta t}
n_c = \frac{V_M \dot{\gamma}_c}{g}
```

## 4. 程序实现说明

本实验没有再把核心解算写成额外函数，主循环直接放在 `main.py` 的 `main()` 内，便于顺着代码逐步核对：

1. 按当前速度比 `P` 计算导弹速度
2. 建立目标与导弹的初始位置
3. 每个积分步先计算当前视线角
4. 按追踪法更新导弹弹道角和位置
5. 同时推进目标位置
6. 记录最大需用过载、最大实际过载、最小弹目距离和命中情况
7. 所有速度比算完后输出图表和结果表

主函数外只保留了三类展示和输出函数：

- 字体配置函数
- 结果保存函数
- 绘图函数

## 5. 运行方式

在本目录执行：

```bash
python main.py
```

## 6. 输出结果

程序运行后会生成：

- `outputs/tables/summary.csv`：速度比扫描结果表
- `outputs/tables/summary.json`：结构化汇总结果
- `outputs/figures/trajectory.png`：不同速度比下的弹道图
- `outputs/figures/metrics.png`：最大需用过载和脱靶量曲线

## 7. 结果关注点

本实验主要看三件事：

- 速度比减小时，弹道是否还能收敛
- 速度比增大后，最大需用过载怎样变化
- 哪些速度比条件下会出现较大的脱靶量或明显过载饱和
